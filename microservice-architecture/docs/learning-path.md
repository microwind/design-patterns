# 学习路径（由简到繁）

## Step 1: 基础拆分

- 从一个业务域拆成 `order-service` / `inventory-service` 两个服务。
- 先用静态地址调用，理解服务边界。

## Step 2: 网关与发现

- 增加 API Gateway 作为统一入口。
- 服务地址由注册中心动态提供。

## Step 3: 通信与负载

- 对比 REST / RPC / MQ 的调用链。
- 引入负载均衡策略（轮询、最少连接、加权）。

## Step 4: 版本与稳定性

- 实现 V1/V2 共存与灰度。
- 增加限流、熔断、超时和降级。

## Step 5: 工程化收敛

- 统一日志追踪、配置管理、测试脚手架。
- 将示例汇总为可复用模板。
