# 微服务架构模式总览

## 为什么补这一类

- 这是现代企业系统的默认架构能力。
- 与“经典设计模式”不同，它关注系统级协作而非单体内对象协作。
- 与“编程范式”互补，强调服务边界、治理、稳定性和演进。

## 模块关系

1. `microservice-basics`：服务拆分、边界、独立部署。
2. `api-gateway`：统一入口与路由策略。
3. `service-discovery`：服务注册与动态发现。
4. `load-balancing`：流量分配与故障摘除。
5. `service-communication`：同步/异步通信选型与实现。
6. `api-versioning`：向后兼容与灰度演进。
7. `resilience-patterns`：限流、熔断、隔离与降级。

## 设计基线

- 先“单一职责服务 + 同步调用”起步。
- 再引入“网关 + 注册发现”。
- 最后叠加“治理能力（版本、限流、熔断）”。
