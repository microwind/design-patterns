## 维度1：代码组织结构

### 代码层次对比

```
PP (过程式)：
函数序列 → 顺序调用 → 全局状态
代码结构简单，易于理解

OOP (面向对象)：
类体系 → 对象交互 → 属性与方法
代码结构复杂，易于扩展

FP (函数式)：
函数组合 → 数据管道 → 纯函数
代码结构优雅，易于测试

AOP (切面)：
核心逻辑 + 切面 → 织入 → 增强代码
代码结构分离，易于维护

EDP (事件驱动)：
事件源 → 事件 → 监听器 → 处理
代码结构解耦，易于扩展

RP (响应式)：
数据源 → Observable → 订阅 → 反应
代码结构流式，易于异步
```

### 代码示例：同一功能实现的代码量

```
需求：计算数组中所有偶数的平方和

PP范式（最少）:
int sum = 0;
for (int num : numbers) {
    if (num % 2 == 0) {
        sum += num * num;
    }
}
// 6行代码

OOP范式：
class Calculator {
    public int calculateSum(int[] numbers) {
        return Arrays.stream(numbers)
            .filter(n -> n % 2 == 0)
            .map(n -> n * n)
            .sum();
    }
}
// 8行代码 + 类定义

FP范式（最优雅）:
numbers
    .filter(isEven)
    .map(square)
    .reduce(0, add)
// 4行代码（不含定义）

AOP范式（注解）:
@Aspect
@Component
public class CalculationAspect {
    @Around("execution(* Calculator.calculate*(..))")
    public Object monitor(ProceedingJoinPoint jp) {
        long start = System.currentTimeMillis();
        Object result = jp.proceed();
        long duration = System.currentTimeMillis() - start;
        logger.info("Execution time: " + duration);
        return result;
    }
}
// 10行切面 + 原方法代码

EDP范式（事件）:
eventBus.on('calculate', (numbers) => {
    const result = numbers
        .filter(n => n % 2 === 0)
        .reduce((sum, n) => sum + n*n, 0);
    eventBus.emit('calculated', result);
});
// 6行代码

RP范式（流）:
Observable.from(numbers)
    .pipe(
        filter(n => n % 2 === 0),
        map(n => n * n),
        reduce((sum, n) => sum + n, 0)
    )
    .subscribe(result => console.log(result));
// 7行代码
```

```
回避：FP, RP, AOP
```
