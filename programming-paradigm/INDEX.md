## 🔗 与其他模块的联系

### 与设计模式的关系
- OOP范式 → GoF设计模式 (23个)
- FP范式 → 函数式设计模式 (高阶函数、组合)
- AOP范式 → 切面设计 (日志、事务、权限)
- EDP范式 → 观察者模式、发布-订阅模式

**相关文档：** [回到设计模式主页](../../)

### 与设计原则的关系
- SOLID原则 → 主要用于OOP设计
- DRY原则 → 代码复用 (所有范式)
- KISS原则 → 简洁性 (PP > FP > OOP)
- YAGNI原则 → 避免过度设计 (EDP最容易违反)

**相关文档：** [设计原则](../../design-principles/)

### 与架构模式的关系
- MVC → OOP范式
- DDD → OOP + FP
- 微服务 → OOP + AOP + EDP
- 响应式架构 → RP范式

## 💡 关键要点速记

### 6大范式一句话总结

| 范式 | 一句话 | 关键词 |
|-----|--------|--------|
| **PP** | 按步骤分解问题 | 顺序、命令式 |
| **OOP** | 模拟对象与交互 | 类、继承、多态 |
| **FP** | 函数组合数据流 | 纯函数、组合 |
| **AOP** | 横切逻辑独立处理 | 切面、织入 |
| **EDP** | 事件驱动流程控制 | 事件、监听 |
| **RP** | 数据流自动响应 | 流、订阅 |

### 快速选择范式

```
"我需要什么？"

性能最优    → PP
代码清晰    → OOP
数据处理    → FP
事务管理    → AOP
模块解耦    → EDP
异步实时    → RP
全要        → 混合使用
```

### 学习难度排序

```
简单 ← PP → OOP → AOP → EDP → FP → RP → 困难
     1周   3月  2月   2月   3月  6月
```

### 性能排序（仅供参考）

```
最快 ← PP → OOP → FP → EDP → AOP → RP → 最慢
     2ms  3ms  4ms  6ms  3ms  8ms
```

## ❓ 常见问题解答

### Q1: 我应该从哪个范式开始学习？
**A:** 从PP开始（基础），然后是OOP（最常用），再学FP（高级思想）

### Q2: 所有范式都要学吗？
**A:** 不必。优先学PP + OOP + FP，其他范式按需学习

### Q3: 哪个范式最难学？
**A:** RP (响应式编程) 最难，需要6-12个月才能掌握

### Q4: 这些范式可以混合使用吗？
**A:** 可以，而且推荐！大多数项目都是多范式混合

### Q5: 我的项目应该用什么范式？
**A:** 查看 [范式选择指南](./PARADIGM_SELECTION_GUIDE.md) 的决策树

### Q6: 学会这些后能做什么？
**A:**
- ✅ 写出更好的代码
- ✅ 做出更好的技术决策
- ✅ 设计更好的系统架构
- ✅ 带领团队提升技术水平
